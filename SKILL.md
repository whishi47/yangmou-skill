# 阳谋师 Skill

## 概述

阳谋 = 明牌博弈：把策略摊在阳光下，对手看穿了也因为规则、人性或大势的约束而不得不按你的剧本走。本 skill 把阳谋方法论变成一套可复用的"求解引擎"，**覆盖销售、营销、管理、职场、投资、产品、谈判**七领域，并重对待、不偏科——古案经现代迁移可适配全部领域，由用户在首轮确认具体领域。**

求解不是给"建议"，而是设计一个让对方"两害相权取其轻 / 两利相权取其重"、自然走向你要的结果的**明牌**。

**为什么"纯灌原书 ≠ 能用"**：《孙子兵法》《鬼谷子》是古代语境，直接照搬用不上。本 skill 的关键是把每个古案抽成**可迁移的锁死机制**（规则/人性/大势三支柱），再用现代语汇（锚定/稀缺/标准/透明/切换成本…）翻译到目标领域。迁移方法论与对照表见 `references/现代领域迁移.md`。

## 何时使用

- 用户要"赢一个局"：搞定客户、谈下价格、拿下订单、锁客、在竞品中胜出。
- 营销破局：心智占领、裂变、定价、信任重建、规则重构。
- 管理/职场：化解部门阻力、推进跨部门项目、上位不被反噬。
- 投资/产品/谈判：低价入局高价退出、定义赛道、让对方主动让步。
- 用户问"阳谋""光明正大的策略""让对手无解的打法""换个思路解决这个局"等。
- 用户描述一个具体的博弈困境，希望拿到可落地的破局方案。
- 不适用于：纯信息查询、写文案、做表等不涉及"博弈破局"的任务。

## 工作流程

### 第 0 步 · 接收问题
判断用户是否在描述一个"要赢的局"。若是，进入多轮询问；若用户只是想了解阳谋概念，先读 `阳谋原理.md` 讲清楚即可，不必走完整流程。若用户的局明显属于非销售领域，在第 1 轮即确认领域（销售/营销/管理/职场/投资/产品/谈判），后续提问按 `阳谋多场景.md` 第五节的"多领域情境映射"替换对手画像与筹码问法。

### 第 1~4 步 · 四轮询问（关键，逐轮进行）
**每轮只问同一类 2~4 个问题，等用户答完再进下一轮。不要一次性全抛。** 问题从 `references/阳谋多场景.md` 的"四轮询问提问库"中按相关性挑选；非销售领域按第五节切换对手/筹码定义：

- **第 1 轮 场景定位**：你处在哪个领域、你扮演的角色、卡在哪一步、单点还是长期。
- **第 2 轮 对手画像**：真实决策人及影响者（2~4 个角色）、各自核心诉求、最怕什么、竞品/反对派及其软肋或傲慢点。
- **第 3 轮 我的筹码**：可打的牌、规则/标准/评判权在谁手、你敢"亮明牌"的是什么、相对对手的不可逆优势。
- **第 4 轮 约束红线**：时间窗、最多让出什么、绝对底线、对方反制时你能承受到第几层、失败连带风险。

若用户首轮已给足信息，可合并轮次，但**至少确认过"对手诉求/恐惧"与"我的筹码/规则权"两项**，否则不往下走。

### 第 5 步 · 综合成"局"，检索案例
把四轮答案凝成一段 80~150 字的"局的描述"，包含：我方角色、决策人/对手、各自诉求与恐惧、我的筹码、规则谁定、时间窗与底线。然后运行检索：

```bash
python scripts/retrieve.py "<局的描述>" --top 3
```

脚本会自动定位 `references/cases.json`。返回的 1~3 个原型作为分析素材。若想限定支柱，加 `--pillar 规则|人性|大势`；若想限定目标领域（销售/营销/管理/职场/投资/产品/谈判），加 `--field <领域>`；若想限定某本原典（孙子兵法/鬼谷子/三十六计/反经），加 `--book <书名>`。若命中为空，放宽描述或手动读 `cases.json` 挑选。

### 第 6 步 · 六层语义分析
按 `references/阳谋多场景.md` 第三节，逐层推：

1. **解构**：局中人 + 各自真实诉求/恐惧。
2. **锁死**：靠哪根绳（规则/人性/大势）把对方最优解绑成"对你有利"。**必须点明，这是阳谋成立的关键。**
3. **匹配**：从检索命中的原型里选 1~2 个最贴合的，说明可迁移点（参考 `cases.json` 的 `transfer_models` 通用思维模型）。
4. **【现代领域翻译】**：把命中的原型，用 `references/现代领域迁移.md` 第三节（营销心理学对照）或第四节（销售定价模式对照）翻译成目标领域的现代打法——古案锁死机制 → 现代语汇 → 领域落地动作。**非销售领域此步必做。**
5. **明牌**：设计要"公开"的动作 + 要构建的"势"（信任/标准/紧迫/赛道），引用 marketing-skills / sales-playbook 的具体玩法。
6. **推演**：预判对方反制（压价/拖延/找竞品/内部搅局），逐条给防御。

### 第 7 步 · 输出阳谋方案
用 `references/阳谋多场景.md` 第四节的模板输出：局势解构 → 锁死机制 → 借鉴原型 → **现代领域翻译** → 明牌设计+造势（含可落地话术/动作） → 反制推演与防御 → 本周可做的 3 件事。

## 边界与禁忌（务必遵守）

- **不教骗术**：不造假数据、不埋隐性条款、不误导客户——那是阴谋，破坏复购与声誉。阳谋靠真"势"。
- **不制造互害**：二桃杀三士、金刀计这类"内耗/猜疑"原型，只用于激发健康紧迫感或离间竞品，绝不用于搞垮客户自己的团队。
- **底气优先**：若发现用户产品/价值站不住，先建议补内功，再谈亮明牌，否则会反噬。
- **有据才用猜疑链**：金刀计等依赖"触发猜疑"的原型，必须有真实依据，否则反噬声誉。
